HOW IT WORKS

Core pipeline

  1. Load: The browser reads the selected file into memory as an ArrayBuffer.
  2. Salt & IV: A cryptographically secure random salt and IV are generated for this file.
  3. Key derivation: PBKDF2-HMAC-SHA-256 derives an AES key from your password and the salt, using an iteration count based on the selected KDF strength.
  4. Encryption: The file contents are encrypted with AES-256-GCM using that key and IV.
  5. Packaging: Salt, IV, and ciphertext are bundled into either a compact binary format or into a self-decrypting HTML file.

Security notes

  • AES-GCM provides both confidentiality and integrity (authenticated encryption).
  • PBKDF2 makes each password guess slower, which raises the cost of brute-force attacks.
  • The page is designed not to upload your file or password through network connections.
