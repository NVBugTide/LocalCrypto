OVERVIEW

Client-side encryption

All operations use the Web Crypto API with qualified encryption. LocalVault works in your local browser and does not send file contents or passwords across the network.

Self-decrypting HTML

You can export a self-contained HTML file that embeds both the encrypted data and the decryption logic. The recipient only needs the file and password to open.

Adjustable KDF cost

PBKDF2-HMAC-SHA-256 is used to derive the encryption key from your password. You can choose iteration counts labelled Normal, Strong, and Paranoid to trade off speed versus brute-force cost.

Focused scope

LocalVault does not try to replace password managers, cloud storage, or backup systems. There is only encrypted payload you can store or share.
